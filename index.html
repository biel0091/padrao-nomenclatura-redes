Padrão de nomenclatura
VSI, VLAN e enlaces do backbone

Referência rápida para nomear VSI de serviço, description de VLAN, enlaces e portas. Consulte antes de criar ou renomear qualquer objeto na rede.

01 · VSI e descrição de VLAN

Dono + nome + UF + serviço + vlan

O nome vem logo depois do dono pra facilitar a busca no Huawei. UF identifica o estado. Serviço fica por último, só pra CLT e PARC.

Para GER, o campo Serviço não entra no nome.
Exemplo de nome
Situação
CLT-NOME-RJ-PPPOE-VLAN
PPPoE do seu cliente, transportada entre OLT e concentrador/BNG
CLT-NOME-SP-L2-VLAN
VLAN do cliente transportada sem mexer no conteúdo (ex: interliga filiais)
CLT-NOME-ES-IP-VLAN
Link IP roteado, entregue direto a um cliente próprio
PARC-NOME-RJ-L2-VLAN
VLAN do parceiro só passando pela sua rede ("carona"), sem você mexer no conteúdo
PARC-NOME-RJ-PPPOE-VLAN
PPPoE de cliente do parceiro, você só transporta
GER-NOME-RJ-VLAN
Gerência e management da infraestrutura
CLT
Cliente próprio, qualquer serviço — inclusive transporte interno (OLT → concentrador).
Pode mexer sozinho
PARC
Tráfego de/para parceiro — você transporta, ele entrega.
Alinhar com o parceiro
GER
Gerência e management (não usa serviço).
Pode mexer sozinho
02 · Enlaces do backbone

Eth-Trunk e porta física

Mesma description para o Eth-Trunk lógico e para cada porta membro. Vale para 10G, 40G e 100G.

  • 01Pontas em ordem alfabética. Evita o mesmo link cadastrado com nomes diferentes em cada lado.
  • 02Número do trunk como diferenciador. Se houver mais de um link físico entre os mesmos dois sites, o TRUNKxx distingue cada um.
  • 03Sem sufixo de porta. A porta membro usa a mesma description do Eth-Trunk — o próprio identificador da interface já mostra qual porta é.
03 · Description de porta física

Dono + nome + modo

Separado do nome da VSI: uma porta trunk pode entregar várias VSIs, e uma VSI pode sair em várias portas — misturar os dois nomes só complicaria.

CLT
Porta que entrega o serviço direto na ponta do cliente próprio, em acesso ou trunk.
Pode mexer sozinho
PARC
Porta de interconexão física com outra operadora.
Alinhar com o parceiro
EQP
Porta que fala com OLT, Mikrotik, uplink de concentrador etc. Não usa modo.
Pode mexer sozinho
  • 01Acesso vs trunk. ACC quando a porta entrega uma única VLAN pro cliente; TRK quando entrega mais de uma VSI na mesma porta.
  • 02EQP sem modo. Porta de OLT, Mikrotik ou outro equipamento normalmente já é trunk por natureza — não precisa repetir no nome.
  • 03Cliente ativado direto na porta, sem VSI. Mesmo padrão CLT-NOME-MODO.
