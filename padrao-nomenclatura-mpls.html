01 · VSI e descrição de VLAN

Padrão de 3 campos

Usado para toda VSI de serviço (cliente, parceiro, transporte, gerência) e para a description correspondente na VLAN.

TIPO–NOME–VLAN
CLT-NOME-VLAN
CLT
Cliente próprio — qualquer protocolo entregue direto ao cliente final (pppoe, ip, l2 dedicado).
Pode mexer sozinho
PARC
Tráfego de ou para outra operadora, mesmo que internamente seja pppoe/ip.
Alinhar com o parceiro
TRP
Transporte interno puro entre POPs do próprio backbone, sem cliente final na ponta.
Pode mexer sozinho
GER
Gerência e management da própria infraestrutura.

02 · Enlaces de 100G

Eth-Trunk e porta física

Mesma description para o Eth-Trunk lógico e para cada porta membro — a própria interface já identifica se é trunk ou porta física.

ENL100G–PONTA_A–PONTA_B–TRUNKxx
ENL100G-PONTA_A-PONTA_B-TRUNKxx
  • 01Pontas em ordem alfabética. Evita o mesmo link cadastrado com nomes diferentes em cada lado.
  • 02Número do trunk como diferenciador. Se houver mais de um link físico entre os mesmos dois sites, o TRUNKxx distingue cada um (ex: TRUNK10, TRUNK20).
  • 03Sem sufixo de porta. A porta membro usa a mesma description do Eth-Trunk — o próprio identificador da interface (ex: GE0/0/1) já mostra qual porta é.
